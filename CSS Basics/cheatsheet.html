01 - Selectors

CSS Selectors & Specificity

Universal Selector *

It works for the entire document, selecting all elements. It is represented by an asterisk (*). When you use the universal selector, it applies the specified styles to every element on the page, unless there are more specific selectors that override those styles for certain elements.

* {
    color: rgb(10, 119, 192);
    background-color: rgb(255, 253, 249);
}
Applies to every element on the page.

Element Selector h1

It targets specific HTML elements by their tag name. For example, if you want to style all <h1> elements or all <button> elements. The styles will apply to all instances of that element, unless more specific selectors override them.

CSS properties are applied in order of specificity. The more specific a selector is, the higher priority it has. The h1 selector is more specific than the universal selector (*), so h1 styles take precedence over universal styles for h1 elements.

h1 {
    color: rgb(42, 42, 42);
}

button {
    background-color: rgb(10, 119, 192);
    color: rgb(255, 255, 255);
}
I am styled by the h1 element selector

ID Selector #id

It targets a specific element based on its unique ID attribute. The ID selector is represented by a hash symbol (#) followed by the ID name. The styles will only apply to the element with the matching ID, and it will override any styles defined in less specific selectors.

The ID selector takes precedence over both the universal selector and the element selector. This is because it is more specific than both.

#innerHeading {
    color: rgb(255, 255, 255);
}
I have a unique ID - highest specificity among these three.

Class Selector .class

It targets elements based on their class attribute. The class selector is represented by a period (.) followed by the class name. The styles will apply to all elements with the matching class and override any styles defined in less specific selectors.

In general we don't give the same ID to multiple elements, but we can give the same class to multiple elements. Also: the color from a class selector will not show if the element selector already sets color - but background-color will show if the element selector does not define one.

.paragraph {
    color: rgb(61, 69, 64);
}

.myClass {
    background-color: rgb(10, 119, 192);
    color: rgb(255, 168, 168);
}
Box A Box B Box C - same class, all styled

03 - Text Properties

Text Properties

text-align

It works according to the parent element - if we apply text-align to a div, all text inside that div will be aligned according to the value. In CSS3 we have two new values: start (aligns left in LTR) and end (aligns right in LTR).

#left   { text-align: left;   }
#center { text-align: center; }
#right  { text-align: right;  }
#start  { text-align: start;  } /* CSS3 */
#end    { text-align: end;    } /* CSS3 */

text-decoration

We can combine multiple values at the same time and also specify the color. A specific use case: if we want to remove the underline from links, we use text-decoration: none.

#underline   { text-decoration: underline; }
#overline    { text-decoration: overline; }
#linethrough { text-decoration: line-through; }

/* combine + color */
#combined { text-decoration: underline overline red; }

/* remove link underline */
#link { text-decoration: none; }

font-weight

We can also specify the font weight using numbers, where 400 is normal and 700 is bold. We can use any number between 100 and 900, where 100 is the lightest and 900 is the boldest.

#normal   { font-weight: normal;  } /* 400 */
#bold     { font-weight: bold;    } /* 700 */
#bolder   { font-weight: bolder;  }
#lighter  { font-weight: lighter; }
#numeric  { font-weight: 600;     } /* any 100–900 */

font-family

We can specify a specific font family (e.g. Arial, Helvetica) or use multiple font families as a fallback mechanism in case the first one is not available. We can use @font-face to define a custom font - we need to specify the font-family name and the source of the font file.

/* generic families */
#serif      { font-family: serif; }
#sans-serif { font-family: sans-serif; }
#monospace  { font-family: monospace; }

/* specific + fallback */
#specific-font {
    font-family: Arial, Helvetica, sans-serif;
}

/* custom font via @font-face */
@font-face {
    font-family: "Softique Demo Bold";
    src: url("SoftiqueDemo-Bold.otf") format("opentype");
    font-weight: normal;
    font-style: normal;
}
#custom-font {
    font-family: "Softique Demo Bold", Arial, sans-serif;
}

04 - Units, Line-Heights & Text-Transforms

Units · Line-Heights · Text-Transforms

Pixels px - Absolute Unit

Pixels (px) represent a fixed size on the screen. 1px is equal to one dot on the screen. It does not scale with the user's settings or the viewport size. Default font size in most browsers is 16px.

#px { font-size: 16px; }
10px 16px (default) 24px 32px

Relative Units - em, rem, %

em - Relative to the font size of the parent element. If the parent has font-size 16px, then 1em = 16px. When the parent's font size changes, 1em changes accordingly.
rem - Relative to the font size of the root element (html). If the root has font-size 16px, then 1rem = 16px. It remains consistent regardless of the nesting level.
% - Relative to the font size of the parent element. If the parent has font-size 16px, then 100% = 16px. Like em, it scales with the parent. 

#em      { font-size: 1em;   }
#rem     { font-size: 1rem;  }
#percent { font-size: 100%;  }
1em (relative to parent) 1rem (relative to html) 100% (relative to parent)

line-height

The line-height property controls the amount of space between lines of text. It can be set using various units (px, em, rem, %). A common practice is to use a unitless value which acts as a multiplier of the font size - e.g. line-height: 1.5 means 1.5 �- the font size.

#lh-normal     { line-height: normal; }
#lh-number     { line-height: 1.5;    }  /* unitless multiplier */
#lh-length     { line-height: 24px;   }
#lh-percentage { line-height: 150%;   }

text-transform

The text-transform property controls the capitalization of text: uppercase - all characters to uppercase. lowercase - all characters to lowercase. capitalize - first character of each word to uppercase. none - no transformation applied. 

#uppercase  { text-transform: uppercase;  }
#lowercase  { text-transform: lowercase;  }
#capitalize { text-transform: capitalize; }
#none       { text-transform: none;       }

The Box Model

Box Model Overview

The content is made up of four parts: content (where text/images display), padding (space between content and border), border (line surrounding padding), and margin (outer space separating element from others).

Width, Height & Border

Using background-color we can see the dimensions of an element. Border shorthand: border: width style color. Full syntax uses three separate properties: border-width, border-style, border-color.

/* shorthand */
.box {
    width: 400px;
    height: 200px;
    border: 2px dotted rgb(136, 17, 17);
}

/* full syntax */
h1 {
    border-width: 2px;
    border-color: rgb(54, 70, 7);
    border-style: solid;
}

border-radius

Used to create rounded corners. The value can be in pixels or percentage. If the value is 50% it will create a circle if the width and height are equal.

#div-a { border-radius: 10px; }
#div-b { border-radius: 25px; }
#div-c { border-radius: 50%;  } /* circle when w == h */

Padding

Padding is the space between the content and the border. The padding can be set for each side (top, right, bottom, left). Shorthand with one value sets all four sides. Shorthand with four values sets T R B L in order.

/* individual sides */
padding-top: 40px;
padding-right: 40px;
padding-bottom: 40px;
padding-left: 40px;

/* shorthand - all four sides */
padding: 40px;

/* shorthand - top right bottom left */
padding: 40px 40px 40px 40px;

08 - Display, Visibility & Alpha Channel

Display · Visibility · Alpha Channel

Block vs Inline

Block elements take up the full width available and start on a new line. Examples: <div>, <h1>, <p>.
Inline elements only take up as much width as necessary and do not start on a new line. Examples: <span>, <a>, <strong>.
We can change the display property of an element to modify its behavior. 

/* block → inline */
#blockChange { display: inline; }

/* inline → block */
#inlineChange { display: block; }
display: block - takes full width, new line
inline A inline B inline C

display: inline-block

Inline elements do not start on a new line, and their vertical margins and padding do not affect the layout. To solve this we can use display: inline-block - similar to inline elements, but with block-level properties like Margin and Padding.

.divEdit1 {
    display: inline;
    /* vertical margin/padding ignored ↑ */
}

.divEdit2 {
    display: inline-block;
    /* respects width, height, margin, padding */
}
inline-block A - has w/h/margin/padding
inline-block B
inline-block C

display: none vs visibility: hidden

display: none - the element is completely removed from the document flow and is not visible.
visibility: hidden - the element is hidden but still takes up space in the layout. 

#box2      { display: none;       }
#box2again { visibility: hidden;  }

Alpha Channel - rgba()

The alpha channel is a component of a color that represents its transparency. It is often used in RGBA (Red, Green, Blue, Alpha) color values. The alpha value ranges from 0 (completely transparent) to 1 (completely opaque).

#alpha2 { background-color: rgba(255, 0, 0, 0.5);  }
#alpha3 { background-color: rgba(255, 0, 0, 0.75); }

em units represent the font size of the element. If the parent font size is 16px, then 1em = 16px. So 2em = 2 �- 16px = 32px.

Root em units mean the font size of the root element (html tag). 1rem = font-size of html. This is consistent regardless of nesting.

vh means a percentage of the viewport height (height of the browser window). vw means a percentage of the viewport width. 50vw = 50% of the browser window width. 50vh = 50% of the browser window height.

11 - Position & Z-Index

Position

position: static (default)

Static positioning means the element is positioned according to the normal document flow. This is the default positioning behavior. We can't change its position in any way - the top, left, right, bottom properties and z-index have no effect.

#box1 {
    position: static;
    top: 20px;    /* no effect */
    left: 20px;   /* no effect */
}
position: static - top/left/right/bottom have no effect here

position: relative

Relative positioning means the element is positioned relative to its normal position. We can change its position using top, left, right, bottom. But it will still take up the same space in the document flow as if it were static.

#box2 {
    position: relative;
    top: 20px;   /* shifted down 20px from its normal spot */
    left: 20px;  /* shifted right 20px from its normal spot */
}
Normal element
position: relative - shifted top:15px left:15px
Next element - space still reserved above

position: absolute

The element is positioned relative to its nearest positioned ancestor (an ancestor with a position other than static). If no such ancestor exists, it is positioned relative to the initial containing block (usually the viewport). It removes the element from the normal document flow, so it does not take up space in the layout.

#box3 {
    position: absolute;
    top: 250px;
    left: 200px;
    /* anchored to nearest positioned parent */
}
Positioned parent (position: relative)
position: absolute - removed from flow, anchored to parent

position: fixed

The element is positioned relative to the viewport (the browser window). It removes the element from the normal document flow so it does not take up space. Commonly used for sticky headers or footers that stay on screen while scrolling.

/* sticky footer bar */
#box4 {
    position: fixed;
    height: 50px;
    width: 100%;
    bottom: 0px;
    left: 0px;
}

/* sticky header bar */
#fixedbox {
    position: fixed;
    height: 50px;
    width: 100%;
    top: 0px;
}
Fixed elements are pinned to the viewport - they don't scroll with the page. Used for nav bars, banners, chat buttons.

position: sticky

The element is positioned relative to the viewport once it reaches a certain scroll position. Before that scroll position it behaves like a relative element.

#box5 {
    position: sticky;
    top: 0;
    left: 50%;
}

11 - Z-Index

Z-Index

Z-Index - Stacking Order

If a div overlaps with another div, the z-index property determines which one appears on top. By default, elements are stacked in the order they appear in the HTML, with later elements appearing on top.

The z-index property only works on elements that have a position value other than static. You need to set position to relative, absolute, or fixed for z-index to take effect.

If the value of z-index is higher, the element will be displayed in front of elements with a lower z-index value. If two elements have the same z-index value, the one that appears later in the HTML will be displayed on top. 

#box1 {
    position: absolute;
    z-index: 1;  /* behind box2 */
}
#box2 {
    position: absolute;
    z-index: 2;  /* in front of box1 */
}
#box3 {
    position: absolute;
    z-index: 3;  /* in front of both box1 and box2 */
}
z-index: 1
z-index: 2
z-index: 3 (top)

Flexbox - Item Properties

order

The box order will be determined by the order of the elements in the HTML. But if we want a custom order, we can use the order property.
1. Boxes with no order or order: 0 appear first, in HTML order. Negative values appear before them.
2. Boxes with positive order values appear after, sorted lowest to highest.
3. If two or more boxes have the same order value, they appear in HTML order. 

#box1 { order: 3;  }
#box2 { order: 1;  }
#box3 { order: 2;  }
#box4 { order: -5; } /* appears first - negative */
#box5 { order: 0;  }
#box8 { order: 5;  }
#box9 { order: 4;  }

flex-grow

Flex grow defines how much space a flex item will take up relative to other flex items. If flex-grow is set to 1, the item will take up all available space in the container. If flex-grow is set to 0, the item will not take up any additional space (default).

.box  { flex-grow: 0; } /* default - no extra space */
.box1 { flex-grow: 1; } /* takes all available space */

flex-shrink

Flex Shrink defines how much a flex item will shrink relative to other flex items when there isn't enough space in the container. If flex-shrink is set to 1, the item will shrink proportionally (default). If flex-shrink is set to 0, the item will not shrink at all and will overflow if there's not enough space and wrap is disabled.

.box2 {
    flex-shrink: 0;
    /* won't shrink - will overflow when space runs out */
}
/* default: flex-shrink = 1 (shrinks proportionally) */
